# StackPilot - 产品需求文档（MVP）

## 一、项目背景

随着 Cursor、VS Code、Trae 等 AI IDE 的普及，开发者越来越频繁地在多个技术栈之间切换：

* Java / Spring Boot
* Go
* Python
* Vue
* React / Next.js
* Node.js
* Rust

为了适配不同技术栈，开发者通常会安装大量插件。

常见问题：

* 插件越来越多
* IDE 启动变慢
* 不知道当前项目需要哪些插件
* 经常忘记切换 Profile
* 新项目环境搭建成本高

StackPilot 希望解决：

> 根据当前项目技术栈，自动识别项目类型，并推荐最合适的 IDE Profile。

补充原则：

* StackPilot 只做识别、推荐和引导
* 不自动安装插件
* 不自动卸载插件
* 不静默切换用户 Profile
* Profile 的创建、导入、切换由用户确认完成

---

# 二、产品定位

StackPilot 是一个 VS Code / Cursor 插件。

核心能力：

* 自动识别项目技术栈
* 推荐最佳 Profile
* 引导用户导入 Profile 模板
* 引导用户手动切换 Profile
* 记住用户选择

不负责：

* 自动生成代码
* 自动安装插件
* 自动卸载插件
* 静默切换 Profile
* AI 编程

这样我无需手动寻找对应配置。

# 五、支持识别的项目类型

## Java

识别条件：

* pom.xml
* build.gradle
* build.gradle.kts

推荐：

Java Profile

---

## Go

识别条件：

* go.mod

推荐：

Go Profile

---

## Python

识别条件：

* requirements.txt
* pyproject.toml

推荐：

Python Profile

---

## Vue

识别条件：

package.json 中存在：

* vue
* nuxt

推荐：

Vue Profile

---

## React

识别条件：

package.json 中存在：

* react
* next

推荐：

React Profile

---

## Node.js

识别条件：

* package.json

且不存在 React/Vue 特征。

推荐：

Node Profile

---

## Rust

识别条件：

* Cargo.toml

推荐：

Rust Profile

---

# 六、交互设计

打开项目后：

检测项目技术栈。

若发现匹配规则：

显示通知：

StackPilot

Detected Spring Boot project.

Recommended Profile:
Java

[Switch]
[Ignore]
[Always Ignore]

---

点击 Switch / Switch Guide：

说明 VS Code 目前没有向插件开放稳定的静默切换 Profile API。

引导用户通过 Profiles 菜单或命令面板手动选择对应 Profile。

如果用户没有对应 Profile，则引导用户导入内置 `.code-profile` 模板。

---

点击 Ignore：

本次忽略。

---

点击 Always Ignore：

记录到本地配置。

后续不再提示。

---

# 七、配置存储

}

# 八、MVP 范围

第一版只实现：

* 项目扫描
* 技术栈识别
* 推荐弹窗
* Profile 模板入口
* 推荐插件目录展示
* 用户选择记录

暂不实现：

* AI 推荐
* 自动安装插件
* 自动卸载插件
* 自动切换 Profile
* 团队同步
* 云配置
* Marketplace 集成
